Gestión / Dirección de Proyectos
* La aplicación de conocimientos, habilidades, herramientas y técnicas a las actividades del proyecto para cumplir con los requisitos del proyecto
* Implica
   * Identificar requisitos
   * Abordar las diversas necesidades, inquietudes y expectativas de los interesados
   * Equilibrar las restricciones contrapuestas del proyecto
      * Alcance
      * Calidad
      * Cronograma
      * Presupuesto
      * Recursos
      * Riesgo


Proyecto
* Esfuerzo temporal que se lleva a cabo para crear un producto, servicio o resultado único
* Con principio y final definidos


Grupos de Procesos
* Iniciar
* Planificar
* Ejecutar
* Seguimiento y Control
* Cierre


Influencias Organizativas y Ciclo de Vida del Proyecto
* -


Influencias Organizacionales
* La cultura, el estilo y estructura de la organización - particular a cada una
   * Visión, misión, valores, creencias y expectativas compartidas
   * Reglamentos, políticas, métodos y procedimientos
   * Sistemas de motivación y recompensa
   * Tolerancia al riesgo
   * Visión de las relaciones de liderazgo, jerarquía y autoridad
   * Código de conducta, ética laboral y horario de trabajo
   * Entornos operativos
* La madurez en la dirección de proyectos facilita la inclusión de entidades externas


Ciclo de Vida de un Proyecto
* -
* Etapas y resultado
   * Inicio → acta de constitución del proyecto
   * Organización y preparación → plan de dirección del proyecto
   * Ejecución del trabajo → entregables aceptados
   * Cierre → documentos del proyecto archivados
  



Grupos de Procesos de Dirección de Proyectos
* Los procesos se aplican globalmente y a todos los grupos de industrias
* El director del proyecto, en colaboración con el equipo del proyecto, determina cuales son los procesos apropiados y el grado de rigor adecuado para cada uno
* Los grupos no son fases del proyecto
  

  



Grupo de Procesos de Iniciación
* Tareas
   * Define el alcance inicial
   * Se comprometen los recursos financieros iniciales
   * Se identifican los interesados que influencian el proyecto
   * Se selecciona el director del proyecto
   * Se genera el acta de constitución del proyecto para su aprobación


Grupo de Procesos de Planificación
* Compuesto por procesos que
   * Establecen el alcance total
   * Definen y refinan objetivos
   * Desarrollan el plan para la dirección y los documentos del proyecto
* Tareas
   * Desarrollar el plan para la dirección del proyecto
   * Alcance
      * Recopilar requisitos
      * Definir el alcance
      * Crear la Estructura de Desglose del Trabajo (EDT)
   * Actividades
      * Definirlas
      * Secuenciarlas
      * Estimar sus recursos
      * Estimar su duración
   * Desarrollar el cronograma
   * Costos
      * Estimar los costos
      * Determinar el presupuesto
   * Planificar la calidad
   * Desarrollar el plan de recursos humanos
   * Planificar las comunicaciones
   * Riesgos
      * Planificar la gestión de riesgos
      * Identificar los riesgos
      * Análisis cualitativo
      * Análisis cuantitativo
      * Planificar la respuesta
   * Planificar las adquisiciones


Grupo de Procesos de Ejecución
* Tareas
   * Dirigir y gestionar la ejecución del proyecto
   * Realizar el aseguramiento de calidad
   * Equipo del proyecto
      * Adquirirlo
      * Desarrollarlo
      * Dirigirlo
   * Comunicación
      * Distribuir la información
      * Gestionar las expectativas de los interesados
   * Efectuar las adquisiciones


Grupo de Procesos de Seguimiento y Control
* Tareas
   * Monitorear y controlar el trabajo del proyecto
   * Realizar el control integrado de cambios
   * Alcance
      * Verificarlo
      * Controlarlo
   * Controlar el cronograma
   * Controlar los costos
   * Realizar el control de calidad
   * Informar el desempeño
   * Monitorear y controlar los riesgos
   * Administrar las adquisiciones


Grupo de Procesos de Cierre
* Tareas
   * Cerrar el proyecto o fase
   * Cerrar las adquisiciones

Desarrollar el Acta de Constitución del Proyecto
* Proceso de desarrollar un documento que autoriza formalmente un proyecto o fase y documenta los requisitos iniciales que satisfacen las necesidades y expectativas de los interesados
* Entradas
   * Enunciado del Trabajo del Proyecto (SOW)
      * Descripción narrativa de los productos/servicios a entregar
      * Referencia a
         * Una necesidad comercial → demanda del mercado, avance tecnológico, requisito legal, regulación del gobierno, etc
         * Una descripción del alcance de un producto
         * Documenta las características del producto/servicio
         * Un plan estratégico
   * Caso de Negocio
      * Proporciona la información necesaria desde una perspectiva comercial para determinar si vale o no la inversión requerida
      * Se crea como resultado de
         * Una necesidad comercial → nueva oportunidad, etc
         * Solicitud de un cliente
         * Requisito legal
   * Contrato
      * (Si el proyecto se está ejecutando para un cliente externo)
   * Factores Ambientales de la Empresa
      * (incluyen: )
      * Las normas gubernamentales / industriales
      * La infraestructura de la organización
      * Las condiciones del mercado
   * Activos de los Procesos de la Organización        
      * (pueden influir: )
      * Los procesos estándar, las normas y las políticas que se utilizan en la empresa
      * Las plantillas → plantilla del acta
      * La información histórica y la base de conocimientos de lecciones aprendidas
* Herramientas y Técnicas
   * Juicio de Expertos
      * Para evaluar las entradas
      * Expertos → todo grupo o individuo con conocimientos o capacitación especializados, internos o externos a la empresa
* Salidas
   * Acta de Constitución del Proyecto
      * (incluye: )
      * Propósito o justificación del proyecto
      * Objetivos medibles y criterios de éxito relacionados
      * Requisitos de alto nivel
      * Descripción del proyecto de alto nivel
      * Riesgos de alto nivel

Dirigir y Gestionar la Ejecución del Proyecto
* Proceso de ejecutar el trabajo definido en el plan para la dirección del proyecto para cumplir con los objetivos del mismo
* Los entregables se producen como salidas de los procesos ejecutados
* Se recopila la información sobre el desempeño del trabajo que se utiliza como entrada para el Grupo de Procesos de Seguimiento y Control
* Requiere la implementación de los cambios aprobados
   * Acción correctiva → para poder alinear el desempeño futuro previsto con el plan
   * Acción preventiva → para reducir la probabilidad de riesgos
   * Reparación de defectos → identificación de un defecto en un componente con una recomendación de reparación o reemplazo del componente


Monitorear y Controlar el Trabajo del Proyecto
* Proceso de monitorear, revisar y regular el avance a fin de cumplir con los objetivos definidos en el plan
* Entradas
   * Informes de desempeño
   * Activos de los procesos de la organización
   * Factores ambientales de la empresa
   * Plan para la dirección del proyecto
* Herramientas y Técnicas
* Salidas
   * Actualizaciones al plan de dirección del proyecto
   * Actualizaciones a los documentos del proyecto
   * Solicitudes a cambio


Realizar el Control Integrado de Cambios
* Proceso de revisar todas las solicitudes de cambio, aprobar y gestionar los cambios en los entregables, en los documentos del proyecto y en el plan para la dirección del proyecto
* Está desde el inicio hasta el final
* Tareas
   * Que solo se implementen los cambios aprobados y gestionarlos
   * Revisar, analizar y aprobar las solicitudes de cambio
   * Mantener la integridad de las líneas base, incorporando solo cambios aprobados al plan y a los documentos
   * Revisar, aprobar o rechazar las acciones preventivas y correctivas recomendadas
   * Coordinar los cambios a través de todo el proyecto
   * Documentar el impacto total de total de las solicitudes de cambio


Cerrar el Proyecto o Fase
* Proceso de finalizar todas las actividades en todos los grupos de procesos de dirección de proyecto
* Entradas
   * Entregables aceptados
   * Activos de los procesos de la organización
   * Plan para la dirección del proyecto
* Salidas
   * Actualizaciones a los activos de los procesos de la organización
   * Transición de producto, servicio o resultado final


Gestión del Alcance del Proyecto
* Incluye los proyectos necesarios para garantizar que el proyecto incluya todo el trabajo requerido para completarlo con éxito
* El objetivo principal es definir y controlar qué se incluye y qué no se incluye en el proyecto


Recopilar Requisitos
* Proceso de definir y documentar las necesidades de los interesados a fin de cumplir con los objetivos
* Entradas
   * Acta de constitución del proyecto
   * Registro de interesados
* Salidas
   * Documentación de requisitos
   * Plan de gestión de requisitos
   * Matriz de rastreabilidad de requisitos


Definir el Alcance
* Proceso de desarrollar una descripción detalla del proyecto y del producto
* Entradas
   * Acta de constitución del proyecto
   * Activos de los procesos de la organización
   * Documentación de requisitos
* Salidas
   * Actualizaciones de los documentos del proyecto
   * Enunciado del alcance del proyecto


Crear la EDT
* (Estructura de Desglose de Trabajo)
* Proceso de subdividir los entregables y el trabajo del proyecto en componentes más pequeños y fáciles de manejar
* Entradas
   * Activos de los procesos de la organización
   * Documentación de requisitos
   * Enunciado del alcance del proyecto
* Salidas
   * Actualizaciones a los documentos del proyecto
   * Línea base del alcance


Verificar el Alcance
* Proceso de formalizar la aceptación de los entregables del proyecto que se han completado
* Entradas
   * Línea base del alcance
   * Entregables validados
   * Documentación de requisitos
   * Matriz de rastreabilidad de requisitos
* Salidas
   * Solicitudes de cambio
   * Entregables aceptados
   * Actualizaciones a los documentos del proyecto


Controlar el Alcance
* Proceso de monitorear el estado del alcance del proyecto y del producto, y de gestionar cambios a la línea base del alcance


Gestión del Tiempo del Proyecto
* Incluye los procesos requeridos para administrar la finalización del proyecto a tiempo


Procesos
* Definir las actividades
   * Identificar las acciones específicas a ser realizadas para elaborar los entregables del proyecto
* Secuencias las actividades
   * Identificar y documentar las interrelaciones entre las actividades del proyecto
* Estimar los recursos de las actividades
   * Estimar el tipo y las cantidades de materiales, personas, equipos o suministros requeridos para ejecutar cada actividad
* Estimar la duración de las actividades
   * Establecer la cantidad de períodos de trabajo necesarios para finalizar cada actividad con los recursos estimados
* Desarrollar el cronograma
   * Analizar la secuencia de las actividades, su duración, los requisitos de los recursos y las restricciones del cronograma
* Crontrolar el cronograma
   * Hacer el seguimiento del estado del proyecto


Gestión de los Costos del Proyecto
* Incluye los procesos involucrados en estimar, presupuestar y controlar los costos de modo que se complete el proyecto dentro del presupuesto aprobado


Procesos
* Estimar los costos
   * Desarrollar una aproximación de recursos financieros necesarios para completar las actividades del proyecto
* Determinar el presupuesto
   * Sumar los costos estimados de actividades individuales o paquetes de trabajo para establecer una línea base de costos autorizada
* Controlar los costos
   * Monitorear la situación del proyecto para actualizar el presupuesto y gestionar cambios a la línea base de costo


Gestión de la Calidad del Proyecto
* Incluye los procesos y actividades que determinan responsabilidades, objetivos y políticas de calidad a fin de que el proyecto satisfaga las necesidades
* Implementa el sistema de gestión de calidad por medio de políticas y procedimientos, con actividades de mejora continua de los procesos llevados a cabo durante todo el proyecto


Procesos
* Planificar la calidad
   * Identificar los requisitos de calidad y/o normas para el proyecto y el producto, documentando la manera en que el proyecto demostrará el cumplimiento con los mismos
* Realizar el Aseguramiento de Calidad
   * Auditar los requisitos de calidad y los resultados de las medidas de control de calidad, para asegurar que se utilicen las normas de calidad apropiadas y las definiciones operacionales
* Realizar el Control de Calidad
   * Monitorear y registrar los resultados de la ejecución de actividades de control de calidad
   * Con el objetivo de evaluar el desempeño y recomendar cambios


Gestión de los RRHH del Proyecto
* Incluye los procesos que organizan, gestionan y conducen el equipo del proyecto
* El equipo está conformado por las personas a las que se le asignaron roles y responsabilidades
* La participación temprana de los miembros del equipo durante el proceso de planificación fortalece su compromiso con el proyecto


Procesos
* Desarrollar el plan de RRHH
   * Identificar y documentar los roles del proyecto, sus responsabilidades, las habilidades requeridas y las relaciones de comunicación
   * Crear el plan para la dirección de personal
* Adquirir el equipo del proyecto
   * Se confirman los recursos humanos disponibles y se forma el equipo
* Desarrollar el equipo del proyecto
   * Se mejoran las competencias, la interacción con los miembros del equipo y el ambiente general para lograr un mejor desempeño
* Dirigir el equipo del proyecto
   * Monitorear el desempeño de los miembros del equipo, dar retroalimentación, resolver problemas y gestionar cambios


Gestión de las Comunicaciones del Proyecto
* Garantiza la generación, recopilación, distribución, almacenamiento, recuperación y disposición final de la información del proyecto
* Una comunicación eficaz crea un puente entre los interesados del proyecto


Procesos
* Identificar a los interesados
   * Identificar a todas las personas u organizaciones impactadas por el proyecto
   * Documentar la información relevante relativa a sus intereses, su participación y su impacto en el éxito del proyecto
* Planificar las comunicaciones
   * Determinar las necesidades de información de los interesados en el proyecto y definir cómo abordar las comunicaciones con ellos
* Distribuir la información
   * Poner la información relevante a disposición de los interesados
* Gestionar las expectativas de los interesados
   * Comunicarse y trabajar en conjunto con los interesados para satisfacer sus necesidades y abordar los problemas conforme se presentan
* Informar el desempeño
   * Recopilar y distribuir información sobre el desempeño, incluyendo informes de estado, las mediciones del avance y proyecciones


Gestión de los Riesgos del Proyecto
* Los objetivos son aumentar la probabilidad y el impacto de eventos positivos, y disminuir la probabilidad y el impacto de eventos negativos


Procesos
* Planificar la gestión de riesgos
   * Definir cómo realizar las actividades de gestión de los riesgos
* Identificar los riesgos
   * Determinar los riesgos que pueden afectar el proyecto y documentar sus características
* Realizar el análisis cualitativo de riesgos
   * Priorizar los riesgos para realizar otros análisis o acciones posteriores, evaluando y combinando la probabilidad de ocurrencia y su impacto
* Realizar el análisis cuantitativo de riesgos
   * Analizar numericamente el efecto de los riesgos identificados sobre los objetivos generales del proyecto
* Planificar la respuesta a los riesgos
   * Desarrollar opciones y acciones para mejorar las oportunidades y reducir las amenazas a los objetivos del proyecto
* Monitorear y controlar los riesgos
   * Implementar planes de respuesta a los riesgos
   * Se rastrean los riesgos identificados, se monitorean los riesgos residuales, se identifican nuevos riesgos y se evalúa la efectividad del proceso de gestión de los riesgos a través del proyecto


Gestión de las Adquisiciones del Proyecto
* Incluye los procesos de compra o adquisición de los productos y servicios que son necesarios fuera del equipo del proyecto
* Incluye los procesos de gestión del contrato y de control de cambios requeridos para desarrollar y administrar contratos u órdenes de compra emitidas


Procesos
* Planificar las adquisiciones
   * Documentar las decisiones de compra especificando la forma e identificando a posibles proveedores
* Efectura las adquisiciones
   * Obtener respuestas de los proveedores, seleccionar uno y adjudicar un contrato
* Administrar las adquisiciones
   * Monitorear la ejecución de los contratos
   * Efectuar cambios y correcciones según sea necesario
* Cerrar las adquisiciones
   * Completar cada adquisición para el proyecto
